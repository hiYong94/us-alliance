# us-alliance

어스얼라이언스 백엔드 엔지니어 채용 과제 — NestJS 기반 Job 관리 시스템.

본 프로젝트의 모든 작업은 다음 세 문서를 단일 진실 공급원으로 삼는다.

- **도메인 · API · 동시성 등 설계 결정** → `docs/requirements.md`
- **코드 작성 규칙 (JSDoc · 주석 · 명명 · 디렉토리)** → `docs/coding-style.md`
- **Git 흐름 (브랜치 · 커밋 · PR · merge)** → `docs/git-convention.md`

각 문서에 명시되지 않은 결정 사항이 발생하면 사용자에게 먼저 확인한 뒤
해당 문서에 추가하여 단일 진실 공급원을 유지한다.
